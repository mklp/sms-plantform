# 企业级高可用短信平台设计说明书

## 1. 项目概述
本项目基于开源项目 SMSGate 进行重构和扩展，旨在构建一个高可用、高并发、支持多租户的企业级短信服务平台。平台采用 Spring Boot 3.5.x 作为核心框架，PostgreSQL 17 作为主数据库，集成中国移动 CMOS 短信核心库 (`sms-core`)，并引入 Redis 和 RocketMQ 提升系统稳定性和削峰填谷能力。

## 2. 架构设计原则
- **高可用性 (High Availability)**: 通过集群部署、数据库主从/流复制、消息队列解耦、熔断降级机制确保服务 99.99% 可用。
- **稳定性 (Stability)**: 核心发送逻辑异步化，失败重试机制，链路追踪，完善的监控告警。
- **多租户隔离 (Multi-tenancy)**: 采用逻辑隔离（TenantID）为主，物理隔离（Schema/Table）为辅的策略，确保企业数据安全和隐私。
- **灵活性 (Flexibility)**: 通道配置、模板管理、签名管理等核心业务参数均支持数据库动态配置，无需重启服务。
- **可扩展性 (Scalability)**: 微服务化模块划分，支持水平扩展。

## 3. 技术栈选型
| 组件 | 技术选型 | 说明 |

| **开发语言** | Java 17+ | 适配 Spring Boot 3.x |

| **配置中心** | Nacos / Apollo | 动态配置管理（可选，本方案优先DB配置） |

## 4. 核心功能模块
1. **租户管理**: 企业注册、密钥管理、配额管理、状态控制。
2. **短信发送**: 
   - 单发/群发
   - 模板短信
   - 变量替换
   - 定时发送
3. **通道管理**: 
   - 多通道接入（移动/联通/电信/第三方）
   - 通道权重轮询/故障自动切换
   - 动态路由策略
4. **回执与上行**: 
   - 状态报告实时回调
   - 用户回复短信处理
5. **统计与报表**: 发送量、成功率、费用统计。
6. **黑白名单与频控**: 全局/租户级频次限制，号码黑名单过滤。

## 5. 数据库设计 (PostgreSQL 17)
### 5.1 多租户策略
采用 `tenant_id` 字段进行逻辑隔离。所有业务表均包含此字段。MyBatis-Plus 拦截器自动注入，防止越权访问。
对于超大型租户，支持配置独立的 Schema。

### 5.2 核心表结构
- **sys_tenant (租户表)**: 存储企业信息、API Key、Secret、余额、并发限制。
- **biz_sms_record (发送记录表)**: 存储发送请求、状态、通道信息、回执时间。使用分区表按时间优化查询。
- **biz_channel_config (通道配置表)**: 存储通道参数（IP、端口、账号、密码）、权重、状态。**核心配置项存于 JSONB 字段，实现灵活性**。
- **biz_template (模板表)**: 审核通过的短信模板。
- **biz_signature (签名表)**: 短信签名管理。
- **sys_dict_config (字典配置表)**: 系统级动态参数（如：默认重试次数、全局开关）。

## 6. 高可用与稳定性设计
### 6.1 异步发送模型
用户请求 -> API 网关 -> 参数校验/频控 -> **写入 RocketMQ** -> 立即返回 MessageID。
消费者组监听 MQ -> 路由选择 -> 调用 `sms-core` -> 发送 -> 更新 DB。
*优势*: 应对突发流量，保护下游通道，解耦业务。

### 6.2 故障转移 (Failover)
- **通道级**: 若某通道连续失败 N 次，自动标记为“不可用”，流量自动切换至备用通道。
- **实例级**: 无状态服务设计，配合 K8s 或负载均衡器实现实例故障剔除。

### 6.3 数据一致性
- 发送记录采用“最终一致性”设计。
- 利用 RocketMQ 事务消息或本地消息表保证“入库”与“发消息”的原子性。
- 定时任务扫描“发送中”状态过久的记录进行补偿重试。

## 7. 灵活性设计
- **配置动态化**: 通道参数、路由规则、频控阈值全部存入数据库。提供管理后台接口修改，内存缓存（Caffeine + Redis）即时刷新，无需重启。
- **SPI 扩展**: 预留 SPI 接口，方便未来接入其他厂商短信核心，不修改核心代码。

## 8. 安全设计
- **认证鉴权**: 基于 API Key/Secret 的签名机制 (HMAC-SHA256)。
- **数据加密**: 敏感配置（如通道密码）在数据库中 AES 加密存储。
- **网络隔离**: 数据库、MQ、Redis 仅内网访问。
